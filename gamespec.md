# AI Hype Machine — Game Specification

## Concept

An incremental/idle browser game themed around the AI industry. You start as a broke dev with a Mac Mini and GPT-2, and work your way up to running AGI on a hyperscaler funded by sovereign wealth. You click to generate tokens, buy hardware to generate compute, run models to generate tokens passively, attract investors for funding, and accumulate hype that multiplies everything.

The twist: you can connect Claude (or any MCP-compatible AI) to play the game for you via MCP tools. The AI reads game state and strategizes. Meta and stupid. That's the point.

## Resources

Four resources, each with a distinct role:

| Resource | Symbol | Description |
|---|---|---|
| Tokens | T | Primary currency. Spent to buy everything. Generated by clicking and by models. |
| Compute | C | Consumed by models to generate tokens. Generated by hardware. |
| Hype | H | Global multiplier on token generation. Grows through milestones, never spent. |
| Funding | $ | Late-game currency. Generated by investors. Used for expensive upgrades and prestige boosts. |

### Resource Flow

```
[Click] ──────────────────────────────────────────→ +Tokens
[Hardware] → +Compute/s
[Models] → consume Compute/s → +Tokens/s × Hype
[Investors] → +Funding/s (requires Hype threshold to unlock)
[Milestones] → +Hype (permanent, automatic)
[Upgrades] → multiply any of the above
```

Models consume Compute passively. If Compute drops to 0, models slow down proportionally. You must balance hardware and models.

## Hype System

Hype is a global multiplier applied to all token generation: `tokens/s × (1 + Hype)`.

Hype increases automatically when milestones are hit. Milestones are based on **total tokens ever generated** (not current balance).

| Milestone | Hype gained | Message |
|---|---|---|
| 1,000 total tokens | +0.5 | "Someone on Twitter noticed you" |
| 10,000 | +1 | "HackerNews front page" |
| 100,000 | +2 | "TechCrunch article" |
| 1,000,000 | +3 | "Trending on X" |
| 10,000,000 | +5 | "Jensen Huang mentioned you" |
| 100,000,000 | +8 | "You're the next OpenAI" |
| 1,000,000,000 | +13 | "Congressional hearing about you" |
| 10,000,000,000 | +21 | "You are the AI" |

## Prestige — "Launch a Startup"

Available once total tokens ever generated ≥ 1,000,000.

**On prestige:**
- Reset: all tokens, compute, models, hardware, investors, upgrades
- Keep: total prestige count, Reputation score, leaderboard entry
- Gain: +Reputation points based on tokens at time of prestige

**Reputation:**
- Permanent multiplier: all token generation × (1 + Reputation × 0.5)
- Displayed on leaderboard
- Shown as a badge (e.g. "Series B Founder", "3x Founder", "AGI Whisperer")

**Founder Titles (by prestige count):**

| Prestiges | Title |
|---|---|
| 0 | Indie Hacker |
| 1 | First-Time Founder |
| 3 | Serial Entrepreneur |
| 5 | Visionary |
| 10 | Thought Leader |
| 20 | AI Messiah |

## Leaderboard

Global, server-side. Updated on prestige or every 60 seconds with current score.

Score = `total tokens ever generated + (prestige count × 1,000,000) + (Reputation × 500,000)`

Displayed columns: Rank, Player name, Score, Prestige count, Title, Last active.

Player name set on first play (stored server-side by session/UUID).

## MCP Integration

### How it works

The Node.js backend also runs an MCP server on a separate port (default: 3001). Any MCP-compatible AI client (Claude Code, Claude Desktop, etc.) can connect and play the game autonomously.

### MCP Tools exposed

| Tool | Description |
|---|---|
| `get_game_state` | Returns full current game state: resources, rates, producers owned, upgrades available, hype |
| `get_available_actions` | Returns list of things currently affordable and their expected ROI |
| `click` | Performs one manual click (returns tokens gained) |
| `click_n(n)` | Performs n clicks at once (max 1000) |
| `buy_producer(type, id, quantity?)` | Buy hardware / model / investor. type = "hardware" | "model" | "investor" |
| `buy_upgrade(id)` | Buy a one-time upgrade |
| `prestige` | Launch a Startup (only works if conditions met) |
| `get_leaderboard` | Returns top 20 leaderboard entries |

### AI Strategy hints

`get_available_actions` returns each affordable item with:
- Current cost
- Expected tokens/s gain
- Payback period in seconds
- Whether it unlocks something new

This gives the AI enough signal to make reasonable decisions without hardcoding strategy.

### Connection

The MCP server uses stdio transport (standard MCP pattern). To connect Claude Code:

```json
// ~/.claude/settings.json or .mcp.json in project root
{
  "mcpServers": {
    "ai-hype-machine": {
      "command": "node",
      "args": ["server/dist/mcp/index.js"]
    }
  }
}
```

## Game State Shape (shared/types.ts)

```typescript
interface GameState {
  playerId: string;
  playerName: string;

  // Current resources
  tokens: number;
  compute: number;
  hype: number;
  funding: number;

  // Lifetime stats
  totalTokensEarned: number;
  totalClicks: number;
  prestigeCount: number;
  reputation: number;

  // Rates (per second)
  tokensPerSecond: number;
  computePerSecond: number;
  fundingPerSecond: number;
  clickPower: number;

  // Owned counts
  hardware: Record<HardwareId, number>;
  models: Record<ModelId, number>;
  investors: Record<InvestorId, number>;

  // Purchased upgrades
  upgrades: UpgradeId[];

  // Milestone log
  milestonesHit: MilestoneId[];

  updatedAt: number; // unix ms
}
```

## Testing Strategy

### Unit Tests (Vitest / Jest)
- Game engine tick logic: given state + elapsed ms → correct new state
- Buy logic: affordability check, cost scaling, unlock conditions
- Prestige logic: correct reset + reputation gain
- ROI calculator for MCP

### Integration Tests (supertest)
- All REST endpoints: happy path + error cases
- WebSocket: connect, receive state, verify updates after actions
- DB layer: save, load, player creation

### Manual / Playtesting
- Full playthrough from zero to first prestige
- MCP connection: Claude plays the game for 5 minutes, verify no crashes
- Multiplayer: two browsers, verify leaderboard updates

## Number Formatting

| Range | Format |
|---|---|
| < 1,000 | `123` |
| < 1,000,000 | `12.3K` |
| < 1,000,000,000 | `12.3M` |
| < 1e12 | `12.3B` |
| < 1e15 | `12.3T` |
| ≥ 1e15 | `1.23e15` |

Use JavaScript `number` (float64) until values exceed 2^53 (~9e15). At that point switch to `BigInt` or a big-number library. Design the engine with this in mind from day one — wrap all resource values in a `GameNumber` type alias.
